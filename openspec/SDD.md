# 系統技術規格書（SDD）— 供應鏈與倉儲物流戰情室

## 1. 功能描述

本系統為倉儲物流管理主管提供即時戰情室儀表板，聚焦五大核心 KPI：庫存週轉天數、缺貨／呆滯預警、出貨時效達成率、揀貨效率、盤差率與損耗。系統依角色權限區分「管理員」與「一般使用者」，管理員可維護資料、調整門檻與儀表板設定；一般使用者僅能檢視儀表板。

### 主要操作流程
1. 使用者於登入頁輸入帳密，經 CheckUserId API 驗證後依角色導向對應畫面。
2. 一般使用者僅檢視儀表板首頁：可篩選倉別／分類，並可點擊圖表（如庫存狀態圓餅圖）連動篩選資料表與 KPI 卡片。
3. 管理員額外可：
   - 於「使用者權限管理」調整角色、鎖定／解除鎖定帳號。
   - 於「物流儀表板設定」管理各 KPI 卡片的顯示狀態與更新頻率。
   - 於「系統設定」調整業務門檻（庫存週轉警戒值、出貨達成率目標、盤差率門檻、登入鎖定次數等）。
   - 於儀表板頁面上傳 Excel 重新繪製庫存資料。
4. 系統於資料超過危險門檻時（如庫存週轉天數 ≥ 危險值），主動跳出 Toast 警示並將對應 KPI 卡片標示為紅色。
5. 使用者可將目前儀表板畫面匯出為 PDF，或個別圖表下載為 PNG 圖檔。

## 2. 資料來源分析表

| 項目 | 來源 | 說明 |
|---|---|---|
| 帳號驗證 | [EXTERNAL] FME-AasApi CheckUserId | 帳號密碼驗證，正式環境由伺服器端代理呼叫，避免瀏覽器直接暴露帳密 |
| 使用者角色／倉別 | [INTERNAL] 本系統自行維護 | 因日翊驗證 API 不回傳角色，角色權限與所屬倉別由本系統資料庫維護 |
| SKU 庫存主檔 | [INTERNAL] 使用者自行維護／Excel 上傳 | 訪談確認：客戶目前以 Excel 維護庫存資料，本系統支援上傳重繪 |
| 出貨紀錄 | [INTERNAL]（待確認：如有 TMS 系統，未來可改為 [EXTERNAL] 串接） | 目前以本系統紀錄或批次匯入 |
| 揀貨紀錄 | [INTERNAL]（待確認：如有 WMS 系統，未來可改為 [EXTERNAL] 串接） | 目前以本系統紀錄或批次匯入 |
| 盤點紀錄 | [INTERNAL] 使用者自行維護 | 定期盤點作業後人工登錄或匯入 |
| 儀表板卡片設定／系統門檻 | [INTERNAL] 管理員維護 | 純系統內部設定資料 |

## 3. 自動計算邏輯表

| 計算欄位 | 公式 | 驗收範例 |
|---|---|---|
| 庫存週轉天數 | Σ目前庫存 ÷ Σ日均出貨量 | 目前庫存 855、日均出貨量 9.5 → 90.0 天 |
| SKU 庫存狀態 | current_stock ≤ safety_stock×30% → 缺貨；daysSinceLastOut > 呆滯門檻(預設45天) 或 daysOfStock > 60 → 呆滯；current_stock ≤ safety_stock → 低庫存；其餘 → 正常 | 現庫存12、安全庫存90 → 12 ≤ 27 → 缺貨 |
| 出貨時效達成率 | 準時出貨筆數 ÷ 總出貨筆數 × 100% | 60筆中58筆準時 → 96.7% |
| 揀貨效率 | Σ揀貨件數 ÷ Σ工時 | 920件 ÷ 8小時 = 115 單/小時 |
| 盤差率 | Σ|實盤-帳面| ÷ Σ帳面庫存 × 100% | 帳面500、實盤486、差14 → 2.8% |
| 損耗金額 | Σ(帳面-實盤 為正的差異 × 單位成本) | 差異14件 × 單價120 → $1,680 |
| 帳號鎖定 | 連續錯誤次數 ≥ 系統設定門檻(預設5次) → 鎖定N分鐘(預設15分鐘) | 連續輸入5次錯誤密碼 → 鎖定至 now()+15min |

## 5. 前端技術棧

- React 18（CDN，供 IT 未來遷移至建置流程 Vite/Next.js 參考）
- Tailwind CSS（CDN，正式上線建議改用 CLI/PostCSS 建置）
- Chart.js 4：庫存週轉趨勢（長條圖＋門檻色帶）、庫存狀態分布（甜甜圈圖，支援點擊聯動）、揀貨效率（分組長條圖）
- SheetJS(xlsx)：Excel 上傳解析
- html2canvas + jsPDF：儀表板匯出 PDF；Canvas.toDataURL：圖表下載 PNG
- RWD：Tailwind 響應式斷點（sm/md/lg），側邊欄於行動裝置收合為漢堡選單抽屜
